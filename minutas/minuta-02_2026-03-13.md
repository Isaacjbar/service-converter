## Acuerdos Tomados

1. Se acordó centralizar la lógica de conversión en un `ConversionService` con caché LRU para optimizar peticiones repetidas.
2. Se definió el uso de `DiagramGenerator` como clase base con patrón Factory para extensibilidad de tipos de diagrama.
3. Se acordó implementar versionado automático en el modelo `DiagramHistory` para rastrear cambios por usuario.
4. Se estableció que el frontend integraría las vistas de historial, upload y diagrama mediante un router protegido con guards de autenticación.
5. Se acordó exponer los endpoints `/convert` y `/examples` en el backend para consumo desde el frontend.
6. Se definió que la Navbar incluiría toggle de tema oscuro/claro como mejora de UX.
